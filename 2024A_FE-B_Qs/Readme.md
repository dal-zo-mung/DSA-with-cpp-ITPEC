**1. _Transition Probability သဘောတရားသုံးပြီး တွက်ချက်ခိုင်းတဲ့ ပုစ္ဆာ_**
[2024A_FE-B_Q1](2024A_FE-B_Q1)
<details>
  
  **၁။ စာလုံးတစ်ခုချင်းစီရဲ့ အဓိပ္ပာယ်**
  
  pa (Stay Cycling): အရင်နှစ်က စက်ဘီးစီးတဲ့သူက ဒီနှစ်မှာလည်း ဆက်ပြီးစီးနေမယ့် နှုန်း။<br/>
  -> _တွက်ပုံ: $1 - pc = 1 - 0.3 = 0.7$ (70%)_ <br/>
  pc  (Stop Cycling): အရင်နှစ်က စက်ဘီးစီးတဲ့သူက ဒီနှစ်မှာ မစီးတော့ဘဲ ရပ်လိုက်မယ့် နှုန်း။<br/>
  ->  _ပေးထားချက်: $0.3$ (30%)_ <br/>
  pb (Start Cycling): အရင်နှစ်က မစီးတဲ့သူက ဒီနှစ်မှာ အသစ်စပြီး စီးလာမယ့် နှုန်း။<br/>
  ->  _ပေးထားချက်: $0.02$ (2%)_ <br/>
  pd (Stay Non-cycling): အရင်နှစ်က မစီးတဲ့သူက ဒီနှစ်မှာလည်း ဆက်ပြီး မစီးဘဲနေမယ့် နှုန်း။<br/>
  ->  _တွက်ပုံ: $1 - pb = 1 - 0.02 = 0.98$ (98%)_


**၂။ ပုစ္ဆာထဲက Matrix P မှာဆိုရင်:**

P[1,1] = pa (Row 1, Col 1) -> စီးရာမှ စီးခြင်း

P[1,2] = pb (Row 1, Col 2) -> မစီးရာမှ စီးခြင်း

P[2,1] = pc (Row 2, Col 1) -> စီးရာမှ မစီးခြင်း

P[2,2] = pd (Row 2, Col 2) -> မစီးရာမှ မစီးခြင်း

**2. _If statement + (Coupon)Logic ပုစ္ဆာ_**
[2024A_FE-B_Q2.cpp](2024A_FE-B_Q2.cpp)
<details>
  
(Q2) ကတော့ ဈေးဝယ်စင်တာတစ်ခုမှာ ကူပွန် (Coupon) ပေးတဲ့ logic ကို မေးထားတာ ဖြစ်ပါတယ်။ 
  
  ပုစ္ဆာက ပေးထားတဲ့ စည်းကမ်းချက် (၂) ခု


**၁။ စည်းကမ်းချက်များ**

Product ID စည်းကမ်း: ဝယ်လိုက်တဲ့ ပစ္စည်းရဲ့ ID နံပါတ်ရဲ့ နောက်ဆုံးဂဏန်းက 3 ဖြစ်ရပါမယ်။ <br/>
(ဥပမာ - Product ID 103, 53, 3 စသဖြင့်)

အရေအတွက် စည်းကမ်း: အပေါ်ကစည်းကမ်းနဲ့ ကိုက်ညီတဲ့ ပစ္စည်း ၃ ခုဝယ်တိုင်း ကူပွန် ၁ စောင် ရပါမယ်။ <br/>
(ဥပမာ - ၆ ခုဝယ်ရင် ၂ စောင် ရပါမယ်)

---

**၂။ Logic ကို ခွဲခြမ်းစိတ်ဖြာခြင်းA အတွက် (Condition စစ်ဆေးခြင်း):**

ပစ္စည်းတစ်ခုရဲ့ နောက်ဆုံးဂဏန်းကို သိချင်ရင် အရှေ့မှာ သင်ခဲ့တဲ့အတိုင်း Modulo 10 (% 10) ကို သုံးရပါမယ်။
ဒါကြောင့် prod_id mod 10 = 3 ဆိုတဲ့ အချက်က မှန်ကန်တဲ့ logic ဖြစ်ပါတယ်။

B အတွက် (ကူပွန်အရေအတွက် တွက်ခြင်း):စည်းကမ်းနဲ့ ကိုက်ညီတဲ့ ပစ္စည်းအရေအတွက် (pur_prod) ကို ၃ နဲ့ စားရင် ကူပွန်အရေအတွက်ကို ရပါမယ်။

ဥပမာ- ၇ ခုဝယ်ထားရင် $7 \div 3 = 2.33...$ ဖြစ်ပေမဲ့ ကူပွန်က ၂ စောင်ပဲ ရမှာဖြစ်လို့ အပိုင်းကိန်းတွေကို ဖြုတ်ချရပါမယ်။
ဒါကြောင့် integer part of (pur_prod ÷ 3) ဆိုတာက အဖြေမှန် ဖြစ်ပါတယ်။
