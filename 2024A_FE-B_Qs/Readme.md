  **၁။ စာလုံးတစ်ခုချင်းစီရဲ့ အဓိပ္ပာယ်**
  
  pa (Stay Cycling): အရင်နှစ်က စက်ဘီးစီးတဲ့သူက ဒီနှစ်မှာလည်း ဆက်ပြီးစီးနေမယ့် နှုန်း။<br/>
  -> _တွက်ပုံ: $1 - pc = 1 - 0.3 = 0.7$ (70%)_ <br/>
  pc  (Stop Cycling): အရင်နှစ်က စက်ဘီးစီးတဲ့သူက ဒီနှစ်မှာ မစီးတော့ဘဲ ရပ်လိုက်မယ့် နှုန်း။<br/>
  ->  _ပေးထားချက်: $0.3$ (30%)_ <br/>
  pb (Start Cycling): အရင်နှစ်က မစီးတဲ့သူက ဒီနှစ်မှာ အသစ်စပြီး စီးလာမယ့် နှုန်း။<br/>
  ->  _ပေးထားချက်: $0.02$ (2%)_ <br/>
  pd (Stay Non-cycling): အရင်နှစ်က မစီးတဲ့သူက ဒီနှစ်မှာလည်း ဆက်ပြီး မစီးဘဲနေမယ့် နှုန်း။<br/>
  ->  _တွက်ပုံ: $1 - pb = 1 - 0.02 = 0.98$ (98%)_

P[1,2] = pb (Row 1, Col 2) -> မစီးရာမှ စီးခြင်း

**3. _Second Largest Element / included(array and if else statement methods)_**
[2024A_FE-B_Q3.cpp](2024A_FE-B_Q2.cpp)
<details>
  
**၁။ Logic ရှင်းလင်းချက်**

အကြီးဆုံးကိန်းကို မှတ်ထားမယ့် max1 နဲ့ ဒုတိယအကြီးဆုံးကို မှတ်ထားမယ့် max2 ဆိုပြီး variable နှစ်ခု ပေးထားပါတယ်။ 
ကိန်းဂဏန်းအသစ် (array[i]) တစ်ခုကို တွေ့တိုင်း အောက်ပါအတိုင်း စစ်ဆေးရမှာပါ။

**အခြေအနေ (A):**<br/>

လက်ရှိကိန်းက အကြီးဆုံးထက် ပိုကြီးနေရင် / တကယ်လို့ array[i] > max1 ဖြစ်ခဲ့ရင်:
အရင် max1 မှာ ရှိနေတဲ့ကိန်းက အလိုအလျောက် ဒုတိယနေရာ (max2) ကို ရောက်သွားပါတယ်။
အခုတွေ့တဲ့ ကိန်းအသစ်က max1 (အကြီးဆုံးနေရာ) ကို ယူလိုက်ပါတယ်။

_A နေရာတွင်: array[i] > max1 ဖြစ်ရပါမယ်။_

**အခြေအနေ (B):** <br/>

လက်ရှိကိန်းက အကြီးဆုံးထက်တော့ မကြီးဘူး / ဒါပေမဲ့ ဒုတိယအကြီးဆုံးထက် ပိုကြီးနေရင်
တကယ်လို့ elseif (array[i] > max2) ဖြစ်ခဲ့ရင်:
ဒီကိန်းက အကြီးဆုံးတော့ မဟုတ်ဘူး၊ ဒါပေမဲ့ လက်ရှိ max2 ထက်တော့ သာနေတဲ့အတွက် max2 နေရာကိုပဲ အစားထိုးလိုက်ရုံပါပဲ။

_B နေရာတွင်: max2 <- array[i] ဖြစ်ရပါမယ်။_

(Q4) ကတော့ ကိန်းဂဏန်းတစ်ခုကို သုဒ္ဓဆခွဲကိန်း (Prime Factors) တွေအဖြစ် ခွဲထုတ်ပေးတဲ့ algorithm ဖြစ်ပါတယ်။ <br/>

<br/><br/>
  
**Logic ရှင်းလင်းချက်**
<br/>
ဒီ algorithm က အငယ်ဆုံး သုဒ္ဓကိန်း i = 2 ကနေ စပြီး စားကြည့်ပါတယ်။

_အခြေအနေ (A):_ <br/>
တကယ်လို့ num ကို i နဲ့ စားလို့ ပြတ်ခဲ့ရင် (အကြွင်း သုည ဖြစ်ခဲ့ရင်) ဒါဟာ ဆခွဲကိန်းတစ်ခု ဖြစ်ပါတယ်။ <br/>
ဒါကြောင့် num ကို i နဲ့ စားပြီး ရလဒ်အသစ်ကို ယူမယ်၊ i ကိုလည်း output ထုတ်ပေးရပါမယ်။ <br/>

ဒါကြောင့် A နေရာမှာ num mod i = 0 ဖြစ်ရပါမယ်။

_အခြေအနေ (B):_ <br/>
ဒီ loop ကို ဘယ်အချိန်အထိ ပတ်နေမလဲ? num တန်ဖိုးက 1 ထက် ကြီးနေသရွေ့ (စားလို့ ရနိုင်သေးသရွေ့) ဆက်ပြီး ပတ်နေရပါမယ်။<br/>
တကယ်လို့ num က 1 ဖြစ်သွားပြီဆိုရင် စားစရာ ဆခွဲကိန်း မရှိတော့လို့ ရပ်လိုက်ရမှာပါ။<br/>

ဒါကြောင့် B နေရာမှာ num > 1 ဖြစ်ရပါမယ်။
